# 2026-02-18 (Tuesday)

## Major Build: Jarvis Prospector 🎯
Built a full LinkedIn Sales Navigator lead generation system in one evening.

### Chrome Extension (`projects/linkedin-prospector/extension/`)
- Manifest V3, content script on Sales Nav pages
- Scrapes lead cards: name, title, company, location, profile URL, connection degree, time in role
- Auto-pagination via URL manipulation + chrome.storage.local persistence
- Campaign dropdown — pick campaign, hit "Open Search" to go to the right Sales Nav search
- Dark theme popup UI matching Matt's other tools
- Multiple pagination fix iterations — DOM button clicking failed, switched to URL-based navigation

### Server API (`projects/linkedin-prospector/server/`)
- Flask on port 8089, systemd service `jarvis-prospector`
- SQLite with leads + campaigns tables
- AI message drafting via Gemini — tailored per campaign type
- 4 pre-seeded campaigns: Local Job Changes, Federal Employees at Risk, Young Families, High Earners
- Full CRUD API for campaigns + leads
- Dashboard with campaign cards, filterable lead table, copy buttons for messages
- Scoring and draft generation endpoints

### How It Works
1. Matt picks a campaign in extension dropdown
2. "Open Search" takes him to Sales Nav with right filters
3. "Start Scraping" grabs all leads from search results
4. "Send to Jarvis" pushes to server for AI scoring + message drafting
5. Dashboard shows leads with ready-to-copy connection requests + follow-up DMs
6. Each campaign has its own messaging style (job change = congrats, federal = empathy, etc.)
